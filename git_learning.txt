
1. add暂存区：
    选择预提交哪些文件，不会有快照；
    可临时撤销覆盖，一键回退工作区

# 暂存指令
git add .     # 暂存所有 update/delete/newbuilt文件
git add -u    # 暂存已跟踪过的文件，忽略新建文件
git add -u path/  # 仅限于该path目录下

# 暂存查看和对比
git status   # 1提交历史  2stage  3workplace
git diff  # 展示具体（暂存vs工作区） : 已change未add的
git diff --cached  # 展示具体（暂存vs上次commit）: 已add未push的

# 增加flag：
git diff --cached --name-status 
git diff --cached --compact-summary


2. commit保存到本地
    暂存区所有内容进行快照snapshot，成为一个版本，存入本地仓库
    永久存在，可回退某版本

# commit指令
git commit -m "message"  # 暂存区所有内容进行快照，存入本地
git commit -a -m "message"  # add后commit前又修改：用-a 等价于补充自动执行 git add -u，先把所有已追踪文件的改动add到stage

# commit查看
git log --oneline  # 所有提交记录
git log origin/main..main --oneline  # 已commit未push -- 对应GUI的「outgoing changed」

# commit对比
git show --name-only HEAD   # 列出commit的文件名; ps:HEAD指代当前版本
git show --name-status HEAD  # 文件+改动类型[A新增/M修改/D删除/R100重命名或改路径+相似度]

ps:
A..B 的含义：存在于 B、但不存在于 A 的所有提交;
A = origin/main，远端的main分支； B = main，本地的main分支

3. push到远端

# push指令
git push [repo_alias] [local_branch] 
git push origin main  -- 远端仓库别名一般是origin，本地的主分支名一般是main

ps:执行过 git push -u origin main / --set-upstream 后，
建立上下游绑定关系： 本地 main ↔ 远端 origin/main
